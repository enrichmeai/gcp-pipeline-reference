QUICK SETUP (ONE COMMAND):

./scripts/create-deployment-sa.sh dev your-project-id

This script will:
  ✅ Create service account

  ✅ Generate JSON key
  ✅ Create Cloud Storage bucket
  ✅ Show commands to add secrets to GitHub
  ✅ Optionally add secrets automatically (using gh CLI)


MANUAL SETUP (IF PREFERRED):
════════════════════════════════════════════════════════════════════════════════

Step 1: Create Service Account & Bucket
────────────────────────────────────────────────────────────────────────────────
PROJECT_ID="your-project-id"

# Create service account
gcloud iam service-accounts create loa-github-deployer-dev \
    --display-name="LOA GitHub Deployer (DEV)" \
    --project=${PROJECT_ID}

# Grant roles
SA_EMAIL="loa-github-deployer-dev@${PROJECT_ID}.iam.gserviceaccount.com"
gcloud projects add-iam-policy-binding ${PROJECT_ID} \
    --member="serviceAccount:${SA_EMAIL}" \
    --role="roles/dataflow.admin"

gcloud projects add-iam-policy-binding ${PROJECT_ID} \
    --member="serviceAccount:${SA_EMAIL}" \
    --role="roles/bigquery.admin"

gcloud projects add-iam-policy-binding ${PROJECT_ID} \
    --member="serviceAccount:${SA_EMAIL}" \
    --role="roles/storage.admin"

gcloud projects add-iam-policy-binding ${PROJECT_ID} \
    --member="serviceAccount:${SA_EMAIL}" \
    --role="roles/iam.serviceAccountUser"

gcloud projects add-iam-policy-binding ${PROJECT_ID} \
    --member="serviceAccount:${SA_EMAIL}" \
    --role="roles/logging.logWriter"

# Create key
gcloud iam service-accounts keys create loa-sa-key-dev.json \
    --iam-account=${SA_EMAIL}

# Create bucket
gsutil mb gs://loa-dataflow-dev-${PROJECT_ID}/


Step 2: Add Secrets to GitHub
────────────────────────────────────────────────────────────────────────────────
# Using GitHub CLI (recommended)
gh secret set GCP_SA_KEY_DEV < loa-sa-key-dev.json
gh secret set GCP_PROJECT_ID_DEV --body "${PROJECT_ID}"
gh secret set GCS_BUCKET_DEV --body "loa-dataflow-dev-${PROJECT_ID}"

# OR via GitHub Web UI:
# https://github.com/YOUR_ORG/legacy-migration-reference/settings/secrets/actions
#   → New repository secret
#   → Name: GCP_SA_KEY_DEV, Value: (paste JSON)
#   → Name: GCP_PROJECT_ID_DEV, Value: your-project-id
#   → Name: GCS_BUCKET_DEV, Value: loa-dataflow-dev-your-project-id

TROUBLESHOOTING:

Issue                          Fix
──────────────────────────────────────────────────────────────────────────────
"Permission denied"            Re-run: ./scripts/create-deployment-sa.sh

"Bucket not found"             gsutil mb gs://bucket-name/
                               gh secret set GCS_BUCKET_DEV --body "bucket-name"

"Invalid credentials"          Re-create key and update GCP_SA_KEY_DEV secret

"Workflow not triggering"      • Check .github/workflows/deploy-loa.yml exists
                               • Push to main or develop branch
                               • Change files in loa_common/ or loa_pipelines/

"Secrets not found"            gh secret list (verify all 3 are added)


SECURITY NOTES:
════════════════════════════════════════════════════════════════════════════════

✅ DO:
  • Delete local key file after uploading: shred -u loa-sa-key-dev.json
  • Rotate keys every 90 days
  • Use only required IAM roles (no Owner/Editor)
  • Enable secret scanning in GitHub repo settings

❌ DON'T:
  • Commit service account keys to Git
  • Echo secrets in workflow logs
  • Share keys via email/Slack
  • Use same SA for production (when you add it later)
